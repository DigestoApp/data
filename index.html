Digesto API Documentation

This document provides information on how to use the Digesto API to retrieve data based on specific dates and editions. The API endpoint is structured as follows:

https://data.digesto.app/{year}/{month}/{day}/{edition}.json

Where:

  • year is a 4-digit year (yyyy).
  • month is a 2-digit month (MM).
  • day is a 2-digit day (dd).
  • edition can be one of the following (times are EST time):
    • morning - Available at 8AM and later.
    • afternoon - Available at 2PM and later.
    • evening - Available at 7PM and later.

API Response Format

The API returns data in a JSON format, structured as follows:

{ "stories": [{ "title": String, "summary": String, "location": String, "category": String, "info": String }] }

This structure includes:

  • stories: An array of story objects, each containing:
    • title: The title of the story.
    • summary: A brief overview of the story.
    • location: The geographical setting or relevance of the story.
    • category: The genre or category of the story.
    • info: Additional reading and citations in Markdown format.

Example Requests

To access the morning edition of March 15, 2023:

https://data.digesto.app/2023/03/15/morning.json

To access the evening edition of July 4, 2023:

https://data.digesto.app/2023/07/04/evening.json

Please ensure your requests are correctly formatted according to the guidelines above. If you encounter any issues or have further questions, please contact our support team.
